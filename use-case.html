ID and name ITR-3: Order one or more tickets for Spider-Man 2
Primary actor User Secondary actor Payroll System
Description An user access the Cinema-city System from the external Internet, searches for the Spider-Man 2 film to watch, selects it and chooses the sits for the tickets. Client orders the tickets and receives an email with the needed data.
Preconditions User must have an account for the app or register
Post-conditions Post-1.Ticket/s ordered saved in the database
Post-2.Sit/s bought by the client are updated to be occupied in the correct room
Normal flow 1.0 User buys a single ticket
1. User opens the Cinema-city app
2. If the user is not already registered, they are prompted to register
3. If they have an account they can log in
4. The user is directed to the homepage containing upcoming movies, browses movies, previous bookings and settings
5. The user browses through the list of available movies and selects Spider-Man 2
6. The app shows a more detailed page about the selected move that includes viewing times
7. the user chooses a viewing time
8. the app presents a siting layout for the selected viewing time
9. the user selects a sit by clicking on the available sits
10. selected sits are highlighted and added to their booking
11. Once the user has selected their desired sits, they proceed by clicking continue (1.1E1)
12. The app shows their booking details for reviewing
13. The user provides payment method and contact information and confirm the booking proceeding payment
14. After successful payment, the user receives a booking confirmation with details like booking ID, movie title, viewing time, selected sits
15. Finally, the user can log out from the application, ending their session
Alternative flow 1.1 User buys more tickets
1. User selects more sits on step 9
2. Continues with step 10
Exceptions 1.1E1 Tow users select the same sit to buy
1. If the users have selected the same sit/sits they are informed
2. The application returns to step 10
ID and name ITR-2: User access the home page and access a movie
Primary actor User
Description An user access the Cinema-city System from the external Internet by logging in and is able to see all the available movies. After he selects a movie he get redirected to a more detailed page
Preconditions User must have an account for the app or register
Post-conditions None
Normal flow 1.0 User access the home page and selects a movie
1. User opens the Cinema-city app
2. If the user is not already registered, they are prompted to register
3. If they have an account they can log in
4. The user is directed to the homepage containing upcoming movies, browses movies, previous bookings and settings
5. The user selects a movie form the available movies
6. The app shows a more detailed page about the selected move that includes viewing times
Alternative flow 1.1 User returns to home page and selects a different movie
1. User clicks on the 'Back' button
2. The app returns to homepage
3. User continues with step 5
ID and name ITR-1: User logs in or registers to homepage
Primary actor User
Description An user access the Cinema-city System from the external Internet by logging in and is able to see all the available movies.
Preconditions User must have an account for the app or register
Post-conditions None
Normal flow 1.0 User logs in
1. User opens the Cinema-city app
2. They log in to the app by pressing 'Login'
3. The user is directed to the homepage containing upcoming movies, browses movies, previous bookings and settings
Alternative flow 1.1 User registers
1. After step 1 the user presses 'Register'
2. The app opens a register page for the user to complete
3. The user gives all the necessary data and presses 'Continue'
4. The app validates the data (see 1.0E1)
5. The app opens homepage
Exceptions 1.0E1 User already exists with the given data
1. The app informs the user that there is already an account with the given data
2. User returns to step 3
